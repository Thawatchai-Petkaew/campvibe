Drop-up: กฎและพฤติกรรม

  • ทิศทางเปิด: ขึ้นบนเสมอ (bottom: calc(100% + 2px)) เพราะ filter row อยู่ใกล้ขอบบน viewport ฝั่งแผนที่ เพื่อไม่ให้เมนูกดทับแผนที่ด้านล่าง
  • ความกว้าง: เต็มความกว้างคอลัมน์ของตัวเอง (stretch) — left:0; right:0 สัมพัทธ์กับคอลัมน์
  • รัศมี: border-radius: 10px 10px 0 0 — โค้งด้านบน ตัดตรงด้านล่างที่ต่อกับ filter row
  • รายการที่เลือก: มีจุดสีเขียว (--teal-glow) ก่อน label + ตัวหนังสือสี teal
  • Keyboard: ArrowUp/ArrowDown เลื่อน, Enter เลือก, Escape ปิด, focus กลับที่ filter trigger
  • คลิกนอก: ปิด drop-up (click-outside dismiss)
  • ARIA: filter trigger = aria-haspopup="listbox" aria-expanded="true/false", เมนู = role="listbox", รายการ = role="option" aria-selected
  • แถว filter ทั้งหมดสูง ≥30px ตลอดเวลา, คอลัมน์เท่ากัน (flex:1), label ตัดด้วย ellipsis ถ้าข้อความยาวเกิน

กฎ truncate + ≥30px: filter-col ต้องสูงขั้นต่ำ 30px และ label ต้องมี overflow:hidden; text-overflow:ellipsis; white-space:nowrap ห้าม wrap เพราะคอลัมน์แคบบนมือถือ ตัวอย่าง: "ผู้ใช้งาน" บนจอ 320px อาจกลายเป็น "ผู้ใช้..."

4Annotations — Specs สำหรับ SMUX-6 Frontend

กฎและค่าที่ Frontend ต้องปฏิบัติตามทุกข้อ

A. Top Bar — Icon-only (<1024px)

  • Breakpoint: <1024px (tablet + mobile) ซ่อน filter dropdowns จาก top bar ทั้งหมด
  • Logo: CampVibe chip ซ้าย (hud-logo-chip) ทำงานเหมือนเดิม ไม่เปลี่ยน
  • ไอคอน 3 ปุ่ม (ขวา):
    • Dashboard → /status : lucide LayoutDashboard, aria-label="ดูผลงานทั้งหมด"
    • Env/Productivity toggle : lucide Gauge, aria-label="ผลผลิต Scout Team"
    • Sound toggle : lucide Volume2 / VolumeX, aria-label="เปิด/ปิดเสียง"
  • Tap target ≥44px: icon button ต้องเป็น w-11 h-11 (44px) ขั้นต่ำ — WCAG 2.1 AA
  • State active: border --glass-border-hi, icon color --teal-glow
  • Tooltip on hover/focus: label Thai สั้นปรากฏด้านล่างปุ่ม (ไม่ใช่ tooltip จาก shadcn ทั่วไป — ใช้ CSS position หรือ Radix Tooltip ก็ได้)
  • Desktop ≥1024: top bar คงเดิมทั้งหมด — ซ่อนไอคอนบน ≥1024, แสดง filter dropdowns + view-link chip เหมือนเดิม
  • Background: top bar ไม่มีพื้น (transparent) เหมือนเดิม
  • Order (ซ้าย→ขวา): Logo → spacer → LayoutDashboard icon → Gauge icon → Volume icon
  • Motion: icon active state ใช้ transition opacity/color 120–250ms, ไม่ animate layout

B. Filter Row (<1024px)

  • Position: ต่อจาก top bar — position:absolute; top:56px; left:0; right:0 (หรือ flex column ตาม implementation); ไม่ใช่ top bar เดิม
  • Layout: 3 คอลัมน์เท่ากัน flex: 1 เต็มความกว้าง frame
  • ความสูง: ขั้นต่ำ 30px — ถ้า label ยาว label ตัด ellipsis แถวไม่ขยาย
  • Label truncate: overflow:hidden; text-overflow:ellipsis; white-space:nowrap บน element label ไม่ใช่บน wrapper
  • Arrow: ▴ (ชี้ขึ้น) สะท้อนว่าเมนูเปิดขึ้นบน (drop-up); เมื่อเมนูปิด ชี้ขึ้นเช่นกัน (convention คง upward เพื่อบอกทิศทาง)
  • Background: --glass border --glass-border — ไม่ใช่ solid สีพื้น
  • Drop-up menu: เปิดขึ้นด้านบนเสมอ, border-radius: 10px 10px 0 0, ความกว้างเท่าคอลัมน์
  • ARIA: aria-haspopup="listbox" aria-expanded aria-controls บน trigger; menu = role="listbox"; item = role="option" aria-selected
  • Keyboard: ArrowUp/Down เลื่อน option, Enter เลือก, Escape ปิด + focus กลับ trigger

C. Mobile Bottom Toolbar — Transparent Bar

  • Bar background: ไม่มีพื้น (transparent) — ลบ background: var(--glass-hi) และ border-top ออกจาก toolbar element
  • ปุ่ม ≡ ทีม / Board: ยังคง glass chip (background: var(--glass); border: 1px solid var(--glass-border)) ในระดับ <button> ไม่ใช่ toolbar
  • Env capsule: อยู่ตรงกลาง (flex:1), ไม่มี background ของตัวเอง — inner ใช้ background: rgba(7,13,28,.65) + border เล็กน้อย
  • Desktop/Tablet ≥1024: bottom toolbar ไม่เปลี่ยน (dock design เดิม)
  • Safe area: padding-bottom ต้องรองรับ iOS safe-area-inset-bottom (env(safe-area-inset-bottom))

D. Env-Pipeline Capsule

  • Data: อ่านจาก MapModel.envLanes.dev, .staging, .prod (array ของ stories); แสดง .length
  • Label mapping: prod lane → label "Ship" (ไม่ใช้ "Prod"/"Production" ใน UI)
  • 3-segment bar widths: proportional — dev/(dev+staging+prod) × 100% ฯลฯ; ถ้าทุก lane = 0 แสดง bar เป็น muted full-width
  • %: projectPct ตรงๆ + ต้องเป็น tabular-nums
  • สี Ship segment: สว่างกว่า Dev/Staging เพื่อให้ "ส่งสำเร็จ" เด่น — ใช้ token --env-ship: #4ade80 (green-400)
  • Tap target capsule: ≥44px height (min-height ของ capsule-inner)
  • Expanded popover: เปิด/ปิดด้วย toggle, dismiss นอก; ใช้ shadcn Popover หรือ custom positioned absolute — ห้าม modal/dialog
Invariant — Night scene immune to .dark: CSS ทั้งหมดใน HUD ของ /status/map ใช้ custom properties ภายใน scope (--night, --glass, --teal-glow ฯลฯ) ไม่ขึ้นกับ token ของ CampVibe light/dark theme รูปแบบนี้ถูกต้องและตั้งใจ ห้าม refactor เป็น bg-card หรือ token theme อื่น

6A11y Checklist — WCAG 2.1 AA

Frontend ต้องผ่านทุกข้อก่อนส่ง design gate

  • Icon buttons (3 ปุ่ม top bar): ทุกปุ่มมี aria-label ภาษาไทย, tap target = 44×44px
  • Filter triggers: aria-haspopup="listbox" + aria-expanded + aria-controls
  • Drop-up items: role="option" + aria-selected; selected item ใช้สี และ dot (ไม่ใช้สีอย่างเดียว)
  • Env capsule: role="button" + aria-expanded + aria-label บรรยายความหมาย (ไม่ใช่แค่ตัวเลข)
  • Summary popover: role="dialog" + aria-label="สถานะ Env Pipeline"; focus ย้ายเข้า dialog เมื่อเปิด
  • Focus ring: visible ring บนทุก interactive element (outline-ring/50 หรือ custom night-scene ring)
  • Keyboard nav: Tab ผ่านปุ่มทุกตัว, ArrowKey ใน drop-up, Escape ปิด menus
  • Color-not-only: Env segments ใช้สี + label text + bar position — ไม่ใช้สีเป็น signal เดียว
  • Contrast: text-night (#F1F6FB) บน glass (rgba 11,30,24,.55–.70) — not measured; Frontend ต้องวัดด้วย axe ก่อน ship
  • prefers-reduced-motion: ปิด shimmer / animation ทั้งหมด; capsule toggle ใช้ opacity snap แทน
  • iOS safe area: toolbar padding-bottom รองรับ env(safe-area-inset-bottom)
