3Narrow Agent Ring — Portrait Layout

The current LAYOUT_NARROW is an alias of LAYOUT_WIDE (same coordinates). On portrait mobile the scale factor (--s) compresses the ring horizontally, pushing agents toward the campfire. The narrow layout uses a tighter oval: same 7 agents, rotationally-even spacing, expanded vertically to use the taller viewport, contracted horizontally to avoid side-panel overlap. No agent overlaps the campfire at any scale.

Wide Ring (current — ≥1024)
Oval aspect ~1.6:1 (wide). Agents well-spaced, no overlap with fire.
Narrow Ring (proposed — <640)
Oval aspect ~0.67:1 (portrait). Tighter x-axis, expanded y-axis. Campfire gap preserved. Agents ≥44px targets. No overlap.

Implementation note for Frontend (SMUX-2): LAYOUT_NARROW at campsite-scene.tsx:159 currently aliases LAYOUT_WIDE. Replace it with the narrow coordinates table above. The existing matchMedia listener at :1244 and :1356–1399 already switches the layout — it just needs different target coordinates in LAYOUT_NARROW. The setHomes() call handles re-positioning. The breakpoint to switch is max-width: 639px (mobile).

4Annotations — Layout Rules + A11y + Motion

The design rules that Frontend must implement in SMUX-2. Every rule here has a rationale.

Breakpoints

  • ≥ 1024px (Desktop): 3-panel layout unchanged. FilterSignposts (3 dropdowns) top · Roster left · Board right · bottom dock.
  • 640–1023px (Tablet): Map fills full width. Filter collapses to 1 compact chip "กรอง ▾". Roster = Sheet side=left (≡ tab on left edge). Board = Sheet side=right (Board ▸ tab on right edge). Bottom dock shrinks to summary only.
  • < 640px (Mobile): Map fills 100vw/100svh. Filter = minimal chip top-right. Roster + Board = Sheet side=bottom. Bottom toolbar (height inside --pad-y: 140px): ≡ ทีม left · stats centre · Board ▸ right.

Sheet Open/Close Contract

  • Open: tap toolbar button → Sheet slides up (translateY 0 → -height, 200ms cubic-bezier(0.23,1,0.32,1))
  • Close triggers: tap × button · Esc key · tap backdrop outside sheet · drag handle down ≥ 80px
  • Focus trap: on open, focus moves to sheet first interactive element; Tab cycles within sheet only; Esc closes and returns focus to the trigger button
  • Backdrop: rgba(0,0,0,.45) overlay behind sheet, z-index below sheet but above map
  • Only one sheet open at a time: opening Board closes Roster and vice versa
  • Keyboard (tablet/desktop): Roster/Board triggers are also keyboard-activatable buttons (not links)

Accessibility (WCAG 2.1 AA)

  • Tap targets ≥ 44px: toolbar buttons min-height: 44px min-width: 44px; agent sprites on mobile — --scout-size clamped to ≥ 44px
  • Keyboard-activatable agents: each agent sprite is role="button" + aria-label="[Role]: [task title or ว่าง]" + tabIndex=0 + onKeyDown Enter/Space
  • Sheet: role="dialog" + aria-modal="true" + aria-label="ทีมงาน" / "Board [epic name]"; focus-trap + Esc + return-focus
  • Toolbar buttons: aria-haspopup="dialog" + aria-expanded state
  • Contrast: #5BE9B0 on rgba(11,30,24,.70) — measured: ~7.2:1 (AA pass). #F1F6FB on glass — ~11:1 (AA pass). Not measured: rgba(223,234,245,.65) muted text — label as not measured, Frontend to verify with axe.
  • Reduced motion: sheet animation disabled; sheets appear instantly. Existing @media (prefers-reduced-motion:reduce) in SCENE_CSS already kills the rAF loop.
  • Loading: map canvas has existing aria-label on scene root (role="img"). Sheet content has role="status" if data-loading.

Motion + CSS Invariants

  • Sheet animation: transform: translateY only (not height/top). Duration 200ms. Easing cubic-bezier(0.23,1,0.32,1) (system motion token). Disabled under reduced-motion.
  • Backdrop: opacity fade 160ms.
  • No Tailwind classes on SCENE_CSS: the scene CSS lives in SCENE_CSS (campsite-scene.tsx) and the dangerouslySetInnerHTML block (page.tsx). Responsive @media rules are added there, not to globals.css. This preserves the immune-to-.dark invariant.
  • Bottom toolbar: inside the existing --pad-y: 140px bottom clearance of .map-stage. No layout shift to the canvas.
  • Sheet CSS: may use Tailwind (shadcn Sheet is a React component) because it is mounted outside the self-contained CSS block via React portal / the App Router layout. The sheet is NOT inside .map-stage.
  • Immune-to-.dark rule: the scene background and all HUD panel colours are hardcoded night-palette values (not design tokens). They do not respond to the global .dark class. This must be preserved.
